Documentação técnica do sistema

1. APIs utilizadas

  • IBGE/SIDRA — Sistema IBGE de Recuperação Automática, fonte oficial do IPCA.
  • Banco Central (SGS) — Sistema Gerenciador de Séries Temporais do Banco Central, que republica em HTTPS a série do IGP-M calculada pela FGV/IBRE. Fonte primária do IGP-M.
  • IPEA Data — base de séries macroeconômicas do Instituto de Pesquisa Econômica Aplicada; também disponibiliza o IGP-M da FGV. Usada como fonte de contingência (fallback).

2. Endpoints consumidos

  • IPCA (tabela 1737, variável 63 — variação mensal): https://apisidra.ibge.gov.br/values/t/1737/n1/all/v/63/p/last%2026/d/v63%202
  • IGP-M — fonte primária, Banco Central/SGS série 189 (últimos 26 pontos): https://api.bcb.gov.br/dados/serie/bcdata.sgs.189/dados/ultimos/26?formato=json. Fallback no IPEA Data (série IGP12_IGPMG12): https://www.ipeadata.gov.br/api/odata4/ValoresSerie(SERCODIGO='IGP12_IGPMG12').

Atenção: tanto o Banco Central quanto o IPEA Data são consumidos via HTTPS, o que elimina o antigo bloqueio de "mixed content" que ocorria com o endpoint HTTP do IPEA ao hospedar a página em HTTPS. Caso alguma dessas APIs não envie os cabeçalhos de CORS necessários, a chamada pode ser bloqueada pelo navegador; nesse cenário, o painel exibe "API indisponível" para o IGP-M sem interromper a exibição do IPCA, e a solução recomendada é intermediar as consultas por um proxy HTTPS próprio (por exemplo, um Cloudflare Worker).

3. Estrutura dos dados retornados

O SIDRA retorna um array em que o primeiro elemento é um cabeçalho descritivo (ignorado pelo sistema) e os demais contêm os campos D3C (competência no formato AAAAMM), D3N (nome do período) e V (valor da variação mensal, em texto). O Banco Central/SGS retorna um array de registros com os campos data (no formato DD/MM/AAAA) e valor (variação mensal, em texto). O IPEA Data (fallback) retorna um objeto OData com a propriedade value, um array de registros contendo VALDATA (data ISO do mês de referência) e VALVALOR (valor numérico da variação mensal).

4. Processo de cálculo do acumulado de 12 meses

Os valores acumulados fornecidos pelas APIs não são utilizados. O sistema calcula o acumulado de 12 meses compondo as 12 variações mensais mais recentes de cada índice através da fórmula ((1+m1/100)×(1+m2/100)×...×(1+m12/100) − 1) × 100, implementada na função calcularAcumulado12Meses(). Caso não existam 12 meses consecutivos válidos para uma competência, o acumulado é exibido como indisponível.

5. Fluxo de atualização do sistema

Ao carregar a página e ao clicar em "Atualizar Índices", a função atualizarIndices() dispara as buscas de IPCA e IGP-M em paralelo, via Promise.allSettled, para que a falha de uma fonte não impeça a renderização da outra. Os dados retornados passam por validarDados() e removerDuplicados(), são combinados em processarSeriesHistoricas() e, por fim, exibidos pelas funções atualizarCards() e atualizarTabela(). A última competência exibida de cada índice é salva em localStorage para destacar, na próxima visita, quando um novo mês for divulgado.

6. Estratégia de tratamento de erros

Toda chamada de rede passa por um wrapper com AbortController e tempo limite, evitando que uma API lenta trave o painel. Falhas são capturadas em blocos try/catch, registradas no console com a função exibirErro() e comunicadas ao usuário por avisos não-bloqueantes na interface — os dados já carregados de outra fonte permanecem visíveis. Lacunas detectadas na série histórica geram um aviso específico, sem interromper a exibição dos meses válidos.

7. Como adaptar o sistema para novos índices econômicos

Para incluir um novo índice (por exemplo, INPC ou SELIC): crie uma função buscarNomeDoIndice() que retorne um array no formato { competencia: 'AAAAMM', valor: number } a partir da API oficial da nova fonte; inclua-a no Promise.allSettled da função atualizarIndices(); estenda processarSeriesHistoricas() para mapear a nova série e calcular seu acumulado de 12 meses; adicione o card correspondente no HTML e as colunas na tabela. Como a validação, a remoção de duplicados e o cálculo de acumulado já são genéricos, nenhuma outra função precisa ser alterada.

Dados oficiais — IBGE/SIDRA (IPCA) e FGV/IBRE via Banco Central/SGS (IGP-M), com IPEA Data como contingência. Sem valores fixos no código; toda consulta é feita em tempo real.
